API REFERENCE GUIDE
===================

BASE URL: http://localhost:5000/api

## AUTHENTICATION ENDPOINTS

### Register User
POST /api/auth/register
Headers: Content-Type: application/json
Body: {
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123"
}
Response: {
  "message": "User registered successfully",
  "user": { "id", "email", "name" },
  "token": "jwt_token_here"
}

### Login User
POST /api/auth/login
Body: {
  "email": "john@example.com",
  "password": "password123"
}
Response: {
  "message": "Login successful",
  "user": { "id", "email", "name", "role" },
  "token": "jwt_token_here"
}

### Logout
POST /api/auth/logout
Response: { "message": "Logout successful" }

## PRODUCT ENDPOINTS

### List Products (Public)
GET /api/products
Query Params:
  - page: number (default: 1)
  - limit: number (default: 12)
  - category: string (optional)
  - search: string (optional)
  - minPrice: number (optional)
  - maxPrice: number (optional)

Response: {
  "products": [...],
  "pagination": { "page", "limit", "total", "pages" }
}

### Get Product Details (Public)
GET /api/products/:id
Response: { product object }

### Create Product (Admin Only)
POST /api/products
Headers: Authorization: Bearer token
Body: {
  "name": "Product Name",
  "description": "...",
  "price": 99.99,
  "category_id": "uuid",
  "image_url": "...",
  "stock": 10
}

### Update Product (Admin Only)
PUT /api/products/:id
Headers: Authorization: Bearer token
Body: { fields to update }

### Delete Product (Admin Only)
DELETE /api/products/:id
Headers: Authorization: Bearer token

## CART ENDPOINTS

### Get User Cart
GET /api/cart
Headers: Authorization: Bearer token
Response: [
  {
    "id": "uuid",
    "product_id": "uuid",
    "quantity": 2,
    "products": { full product object }
  }
]

### Add to Cart
POST /api/cart/add
Headers: Authorization: Bearer token
Body: {
  "product_id": "uuid",
  "quantity": 1
}

### Update Cart Item
PUT /api/cart/:id
Headers: Authorization: Bearer token
Body: { "quantity": 3 }

### Remove from Cart
DELETE /api/cart/:id
Headers: Authorization: Bearer token

### Clear Cart
DELETE /api/cart
Headers: Authorization: Bearer token

## ORDER ENDPOINTS

### Create Order
POST /api/orders/create
Headers: Authorization: Bearer token
Body: {
  "total_amount": 199.99,
  "items": [
    { "product_id": "uuid", "quantity": 2, "price": 99.99 }
  ],
  "shipping_address": "123 Main St, City, ZIP",
  "payment_method": "card"
}
Response: { order object }

### Get User Orders
GET /api/orders/user
Headers: Authorization: Bearer token
Response: [
  {
    "id": "uuid",
    "total_amount": 199.99,
    "status": "pending",
    "payment_status": "pending",
    "order_items": [...]
  }
]

### Get Order Details
GET /api/orders/:id
Headers: Authorization: Bearer token
Response: { order object with items }

### Update Order Status (Admin Only)
PUT /api/orders/:id
Headers: Authorization: Bearer token
Body: {
  "status": "shipped",
  "payment_status": "completed"
}

## ERROR RESPONSES

All errors follow this format:
{
  "error": "Error message",
  "status": 400
}

Common status codes:
- 200: Success
- 201: Created
- 400: Bad Request (validation error)
- 401: Unauthorized (no token or invalid token)
- 403: Forbidden (insufficient permissions)
- 404: Not Found
- 500: Server Error

---

## AUTHENTICATION

All protected endpoints require Authorization header:
Headers: Authorization: Bearer {jwt_token}

Token obtained from:
- /api/auth/register (during registration)
- /api/auth/login (during login)

---

## EXAMPLE CURL COMMANDS

Register:
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"John","email":"john@test.com","password":"pass123"}'

Login:
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"john@test.com","password":"pass123"}'

Get Products:
curl http://localhost:5000/api/products

Get Products with Filter:
curl "http://localhost:5000/api/products?category=electronics&minPrice=10&maxPrice=100&page=1"

Get Cart (with auth):
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:5000/api/cart

Add to Cart:
curl -X POST http://localhost:5000/api/cart/add \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"product_id":"uuid","quantity":1}'

## RATE LIMITING

All API endpoints are rate-limited:
- Window: 15 minutes
- Max requests: 100 per IP

Rate limit headers in response:
- X-RateLimit-Limit: 100
- X-RateLimit-Remaining: 99
- X-RateLimit-Reset: timestamp

---

## PAGINATION

For endpoints that support pagination:
- Default limit: 20
- Max limit: 100
- Page starts from 1

Response include pagination info:
{
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 50,
    "pages": 3
  }
}

To get page 2:
/api/products?page=2&limit=20

---

## SORTING & FILTERING

Products can be filtered by:
- category: Category ID
- minPrice: Minimum price
- maxPrice: Maximum price
- search: Product name search

Example:
/api/products?category=123&minPrice=10&maxPrice=500&search=laptop

---

## TIMESTAMPS

All resources include:
- created_at: ISO 8601 format
- updated_at: ISO 8601 format

Example: 2024-01-01T00:00:00.000Z
