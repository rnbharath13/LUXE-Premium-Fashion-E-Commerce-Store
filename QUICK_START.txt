QUICK START GUIDE
=================

## 1. INSTALL DEPENDENCIES

### Backend
cd server
npm install

### Frontend (root)
npm install

## 2. SETUP SUPABASE

1. Create account at https://supabase.com
2. Create new project
3. Go to SQL Editor
4. Run SQL from server/models/schema.js
5. Copy your credentials:
   - Project URL → SUPABASE_URL
   - Anon Key → SUPABASE_ANON_KEY
   - Service Role Key → SUPABASE_SERVICE_KEY

## 3. CREATE ENV FILES

### server/.env
PORT=5000
NODE_ENV=development
SUPABASE_URL=paste_your_url
SUPABASE_ANON_KEY=paste_your_key
SUPABASE_SERVICE_KEY=paste_your_key
JWT_SECRET=your_secret_key_here
JWT_EXPIRE=7d
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100

### .env (root)
VITE_API_URL=http://localhost:5000/api

## 4. START SERVERS

### Terminal 1 - Backend
cd server
npm run dev
# Should print: Server running on port 5000

### Terminal 2 - Frontend
npm run dev
# Should print: Local: http://localhost:5173/

## 5. TEST THE APPLICATION

1. Open http://localhost:5173/ in browser
2. Register new account
3. Login with credentials
4. Browse products
5. Add to cart
6. Checkout (orders won't work without payment integration)

## 6. ADMIN ACCESS

To make a user admin, manually update in Supabase:
1. Go to Supabase → Table Editor → users
2. Find your user
3. Change role from 'customer' to 'admin'
4. Now access /admin/dashboard

## 7. BUILD FOR PRODUCTION

### Frontend
npm run build
# Creates dist/ folder

### Backend
# Ready to deploy to Render/Railway/Heroku

## IMPORTANT FILES

Backend:
- server/server.js - Express setup
- server/controllers/ - Business logic
- server/routes/ - API endpoints
- server/models/schema.js - Database schema

Frontend:
- src/App.jsx - Main routing
- src/context/ - Global state
- src/pages/ - Page components
- src/components/ - Reusable components
- src/services/api.js - API calls

## COMMON ISSUES

❌ Port 5000 already in use?
   Kill: npx kill-port 5000

❌ CORS errors?
   Check server/.env FRONTEND_URL matches http://localhost:5173

❌ Supabase connection fails?
   Verify .env file has correct credentials

❌ Products not showing?
   Ensure SQL schema is created in Supabase

## API TESTING WITH CURL

Register:
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"password123","name":"John"}'

Get Products:
curl http://localhost:5000/api/products

Login:
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"password123"}'
